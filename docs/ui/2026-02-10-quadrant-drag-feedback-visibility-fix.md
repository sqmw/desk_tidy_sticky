# 四象限拖拽反馈可见性修复（2026-02-10）

## 背景
- 用户反馈四象限拖拽“没有任何效果”，实际拖拽事件存在，但视觉反馈在当前主题下过弱。

## 修复点
- 拖拽开始/结束时，给 `body` 增加/移除 `ws-note-dragging` 状态类，统一拖拽态。
- 四象限容器新增 `drag-active` 状态：
  - 拖拽中所有象限边框增强高亮，提示可投放区域。
- 被拖拽卡片强化反馈：
  - 透明度降低、虚线边框、高亮背景、`拖动中` 标签。
- 投放指示条强化：
  - 增高、加粗、对比度提升、外圈发光，避免浅色/深色主题下不明显。

## 影响文件
- `src/lib/components/panel/WorkbenchSection.svelte`

## 结果
- 拖拽过程可见性显著提升，用户能明确看到：
  - 当前正在拖哪张卡；
  - 当前可落在哪个象限；
  - 具体将插入到哪一行。
