# 番茄计时 UX 第三轮（绑定可视化/继续按钮/统计摘要）（2026-02-10）

## 目标
按体验优先完成 3 项优化：
1. 绑定状态可视化（当前任务进度）
2. 暂停后按钮语义清晰（继续）
3. 统计折叠态保留摘要信息

## 实现
1. 计时卡增加“当前任务番茄进度”胶囊：
   - 展示 `已完成 / 目标`
   - 展示进度条
2. 主按钮文案逻辑：
   - 运行中：`暂停`
   - 已开始但暂停：`继续`
   - 未开始：`开始`
3. 统计折叠按钮增加摘要：
   - 今日专注分钟
   - 连续专注天数

## 2026-03-27 状态更新（二）
- 判定：`设计收敛`
- 顶部计时卡不再承担“任务选择 / 快捷时长”职责。

### 调整
1. 删除顶部 `25m / 40m` 快捷档。
2. 删除顶部任务下拉框。
3. 顶部仅保留当前任务信息、倒计时和进度条。
4. 原顶部 `设置` 已下沉到 `专注任务规划` 区，改为更轻的 `编辑设置` 入口。

### 原因
1. 任务已经在下方 `专注任务规划` 中创建并启动。
2. 顶部再放一套任务选择与时长入口，会形成双入口，增加认知负担。
3. 顶部再放一个大 `设置` 按钮，会与下面的任务编辑形成认知混淆。
4. 收口后，顶部是“运行控制区”，下方是“任务规划/编辑区”，职责更清晰。

## 2026-03-27 状态更新（三）
- 判定：`设计收敛`

### 调整
1. 顶部计时卡移除 `暂停` 与 `重置`，不再提供运行态操作按钮。
2. 任务行主按钮改为运行态按钮：
   - 未开始：`开始`
   - 运行中：`暂停`
   - 已暂停：`继续`

### 原因
1. 顶部和任务行同时提供暂停入口，会形成重复控制。
2. 既然任务是从任务行启动的，那么暂停/继续也应当在同一位置完成，交互更闭环。
3. `重置` 属于低频破坏性操作，继续留在顶部会打断“任务行即控制点”的单一心智模型。

## 2026-03-27 状态更新（四）
- 判定：`设计问题`

### 调整
1. 番茄钟模式下，顶部计时卡改为按内容自适应高度。
2. 休息控制模式继续保留较大的高度预算，用于承载两列配置内容。
3. 番茄钟模式进一步压缩内边距、信息卡间距和进度条上边距，避免切回番茄钟时出现“卡片虽不空但仍显松散”的观感。

### 原因
1. 在移除顶部任务选择、快捷时长和运行按钮后，番茄钟模式只剩倒计时、进度条和摘要信息。
2. 原来的统一 `min-height` 会把番茄钟卡片撑成大面积空白，视觉重心被破坏。
3. 两个模式的内容密度已经明显不同，不应继续共享同一套高度预算。
