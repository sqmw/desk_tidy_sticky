# Workspace 卡片标签菜单增强（2026-02-11）

## 问题

卡片上的 `Q1~Q4` 菜单只支持四象限切换，无法直接复用已存在标签，也不能在弹层内快速新建标签。

## 改动

在工作台卡片的标签菜单中增加“标签能力”：

1. 保留系统标签项：
   - 无标签
   - Q1 / Q2 / Q3 / Q4
2. 新增“已使用标签”区：
   - 从当前列表中聚合常用标签（按频次排序）
   - 点击即可给当前卡片增/删该标签（toggle）
3. 新增“新建标签”输入：
   - 输入后回车或点击按钮即可创建并绑定到当前卡片
   - 自动去除前缀 `#` 与空白

## 涉及文件

- `src/lib/components/panel/WorkbenchSection.svelte`
  - 增加标签聚合、toggle、新建逻辑
  - 扩展菜单 UI 与样式
- `src/routes/workspace/+page.svelte`
  - 透传 `updateTags` 给工作台卡片区

## 设计说明

为保持稳定，底层依旧是：

- 系统标签：`priority`（Q1~Q4）
- 自定义标签：`tags[]`

交互层已统一用“标签”心智呈现，后续可再做底层字段彻底合并迁移。

## 2026-03-26 补充：标签菜单按钮文案纠正

- 判定：这是 Bug/回归，不是设计行为。
- 根因：工作台卡片标签菜单底部创建按钮误复用了 `workspaceCreateNote`，导致显示成“新建笔记”。
- 修复：新增独立文案键 `workspaceCreateTag`，按钮仅用于“新建标签 / Create tag”。
- 代码：
  - `src/lib/components/panel/WorkbenchSection.svelte`
  - `src/lib/strings.js`
- 回归：
  - 标签菜单输入框右侧按钮应显示“新建标签”
  - 不影响顶部真正的“新建笔记”入口

## 2026-03-26 补充：标签创建区宽度预算重排

- 判定：这是设计问题，不是逻辑 Bug。
- 根因：原布局使用双列 `input + auto button`，在窄菜单里按钮文案会直接挤压输入框，导致标签输入区退化成极小宽度。
- 修复：
  - 标签菜单改为固定宽度预算：`176px ~ 240px`
  - 创建区改为上下布局：第一行输入框占满，第二行按钮独立
  - 标签列表宽度改为跟随菜单，不再单独限制 `220px`
- 代码：
  - `src/lib/components/panel/WorkbenchSection.svelte`
- 回归：
  - 标签输入框应始终有可输入的完整宽度
  - “新建标签”按钮不再压缩输入框
  - 两处标签菜单（列表卡片 / 象限卡片）行为保持一致

## 2026-03-26 补充：标签菜单脱离滚动容器裁剪

- 判定：这是 Bug/回归，不是设计行为。
- 根因：
  - 原标签菜单直接挂在卡片内部的 `.priority-wrap`
  - 工作台列表容器 `.workbench` 使用 `overflow: auto`
  - 菜单一旦越出滚动容器可视区，就会被裁掉；这不是 `z-index` 能解决的问题
- 修复：
  - 菜单改为单实例视口浮层，不再作为卡片内部子元素渲染
  - 打开菜单时记录触发按钮位置，按视口坐标计算浮层 `top/left/width`
  - 滚动工作台或窗口尺寸变化时，实时重算菜单位置
  - 点击菜单外关闭逻辑同步兼容浮层层级
- 代码：
  - `src/lib/components/panel/WorkbenchSection.svelte`
- 回归：
  - 靠左/靠右/靠上边界打开标签菜单时，不应再被裁剪
  - 列表视图与四象限视图使用同一份浮层菜单逻辑

## 2026-03-26 补充：标签输入框 Backspace 光标跳首位

- 判定：这是 Bug/回归，不是设计行为。
- 根因：标签菜单改为单实例浮层后，输入值更新会驱动组件刷新，浏览器在该路径下未稳定保留 caret 位置，表现为按 `Backspace` 后光标跳到首位。
- 修复：标签输入框改为显式 `oninput` 同步草稿，并在刷新后恢复输入焦点与选择区间，确保 caret 留在用户当前编辑位置。
- 代码：
  - `src/lib/components/panel/WorkbenchSection.svelte`
- 回归：
  - 在标签输入框中连续输入/回退时，光标不应再跳到首位
  - 不影响 Enter 创建标签
