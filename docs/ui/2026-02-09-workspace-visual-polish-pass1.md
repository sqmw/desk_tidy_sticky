# 2026-02-09 工作台视觉优化（Pass 1）

## 触发背景
根据反馈与截图，当前工作台存在明显体验问题：
1. 动作按钮使用字母占位（`E/D/P...`），语义弱且不专业。
2. 卡片预览容易被长内容/图片污染，视觉噪声高。
3. 工具栏与侧栏层级弱，整体看起来像“放大版简洁模式”。

## 本次优化目标
1. 将动作按钮改为图标语义，降低理解成本。
2. 统一卡片密度与预览边界，避免内容失控。
3. 强化工作台视觉层级，使其与简洁模式有明显差异。

## 具体实现
### 1) 卡片动作按钮重构（图标化）
- 文件：`src/lib/components/panel/WorkbenchSection.svelte`
- 变更：
  - 便笺操作从字母按钮改为图标按钮（编辑、完成、归档、置顶、层级等）。
  - 保留 `title` 文案，兼顾可发现性与无障碍提示。
  - 优先级按钮保留 `Q1~Q4` 文本，突出任务语义。

### 2) 卡片预览边界治理
- 文件：`src/lib/components/panel/WorkbenchSection.svelte`
- 变更：
  - 卡片统一 `min-height`，悬浮态增加轻微抬升与阴影。
  - 正文区域设置渐隐遮罩，避免长文“硬截断”突兀。
  - 图片预览增加尺寸上限与圆角边框（`max-height` + `object-fit`）。
  - `pre/table` 等复杂块做压缩显示，减少卡片挤压。

### 3) 工作台外层样式增强
- 文件：`src/routes/workspace/+page.svelte`
- 变更：
  - 背景改为多层渐变与氛围光斑，提升“工作台”识别度。
  - 左侧栏增加品牌徽标、分组区块、主次按钮样式。
  - 顶部工具栏改为玻璃卡片化容器（边框/圆角/模糊）。
  - 视图按钮增加 hover/active 态，提升操作反馈。

## 验证
1. `npm run check`：通过（0 errors / 0 warnings）
2. `cargo check`：通过（仅既有 warning）

## 后续建议（Pass 2）
1. 把工作台卡片按类型拆分（text/todo/media）并使用专用渲染策略。
2. 为工作台加入可配置密度（舒适/紧凑）与卡片尺寸偏好。
3. 将 `workspace` 的 sidebar 与 toolbar 抽为独立组件，进一步降低页面耦合。
