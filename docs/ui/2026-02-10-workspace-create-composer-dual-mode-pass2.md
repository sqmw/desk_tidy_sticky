# Workspace 创建区双模式优化（Pass2）

日期：2026-02-10  
范围：`workspace` 模式下的顶部新建输入区（不涉及拖拽逻辑）

## 背景
- 现状是单行输入 + 两个按钮，适合短便签，不适合长内容起草。
- 用户在“快捷记录”和“长文档”之间缺少明确的输入心智切换。

## 本次改动
### 1) 新增创建模式切换
- 在创建区增加模式分段按钮：
  - `快捷记录`
  - `长文档`
- 作用：把“快速输入”和“长文本起草”从一个输入形态中解耦，减少误操作。

### 2) 输入控件按模式切换
- `快捷记录`：单行 `input`
  - `Enter` 直接创建。
- `长文档`：多行 `textarea`
  - 支持换行输入；
  - `Ctrl+Enter` / `Cmd+Enter` 创建；
  - 显示 `取消` 按钮，取消会清空草稿并回到快捷模式。

### 3) 文案与可发现性
- 新增多语言文案键：
  - `workspaceCreateModeLabel`
  - `workspaceCreateModeQuick`
  - `workspaceCreateModeLong`
  - `workspaceLongDocHint`
- 目标：让用户在无需学习成本的情况下理解当前输入行为。

## 设计取舍
- 没有新增全局状态或后端字段，保持最小变更面。
- 保留原有 `onSave` / `onCreateLongDoc` 接口，避免影响其他模块。

## 验证
- 执行：`npm run check`
- 结果：`0 errors, 0 warnings`

## 后续建议
- 若要继续提升长文档体验，可在 `长文档` 模式加入：
  - 最近草稿恢复；
  - 模板快速插入（如会议纪要、学习笔记）。
