# Workstation 四象限占位“卡在上一个位置”修复（2026-02-10）

## 现象
- 用户把拖拽卡片移动到下一张卡片上方时，占位仍停留在上一个位置。

## 根因
- 插入索引按“指针点”计算，而用户视觉上是按“整张拖拽卡片位置”判断。
- 当指针位于卡片左上拖拽手柄附近时，指针 Y 与卡片主体中心存在明显偏差。

## 修复
- 改为使用拖拽浮层的“可视探针点”计算命中与插入：
  - `probeX`: 浮层宽度 30% 位置（避免贴边误判）
  - `probeY`: 浮层垂直中心
- 象限命中与卡片插入索引统一使用 `probeX/probeY`。

## 影响文件
- `src/lib/components/panel/WorkbenchSection.svelte`

## 验证
- `npm run check` 通过（0 error / 0 warning）。
