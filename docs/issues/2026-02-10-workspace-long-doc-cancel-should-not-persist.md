# 写长文档后点取消仍然创建记录

## 现象
- 在工作台点击“写长文档”后进入详情编辑。
- 用户点击“取消”或“关闭”后，仍保留了一条新建记录，不符合“取消即不创建”的预期。

## 根因
- “写长文档”会先创建一条真实笔记，再进入编辑态。
- 取消路径仅切换 UI 状态（`edit -> view`），未回滚这条新建记录。

## 修复
- 文件：`src/routes/workspace/+page.svelte`
- 增加“临时长文档草稿”标记：
  - `pendingLongDocDraft = { id }`
- 行为调整：
  1. “写长文档”创建后标记为临时草稿。
  2. 若该草稿在详情页点击“取消”或“关闭”：
     - 调用 `permanently_delete_note` 撤销；
     - 重新加载列表并同步贴纸窗口；
     - 关闭详情面板。
  3. 若保存成功，则清除临时草稿标记，不再回滚。

## 验证
- 执行：`npm run check`
- 结果：`0 errors, 0 warnings`
