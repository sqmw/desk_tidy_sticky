# 2026-02-01 — 贴纸独立窗口：顶部边界偏移（误把 Header 计入可视上边界）

## 问题
- 在“每个贴纸一个独立窗口”（`NoteWindowPage`）模式下，把窗口拖到屏幕最上方时，贴纸内容顶部会“空出一条透明间隔”，看起来像是被限制了位置，无法贴到屏幕上边界。
- 这个间隔的高度约等于贴纸 Header 高度（`32px`），但 Header 在非悬浮/非编辑时是隐藏/忽略的（`height: 0` + `IgnorePointer`），因此不应影响“贴纸可视内容”的上边界。

## 根因
- `NoteWindowPage` 的窗口高度估算一直包含 `Header(32px)`，用于保证悬浮/编辑时按钮区域有空间。
- 同时页面使用 `Align(alignment: Alignment.bottomCenter)` 把 `StickyNoteCard` 贴到窗口底部：
  - 当 Header 隐藏（卡片高度减少）时，窗口顶部会留下额外的透明空间；
  - Windows 又会限制窗口本身不容易/不允许继续往上拖出屏幕（窗口 `y < 屏幕 top`），于是用户看到的“贴纸内容”就永远差一个 Header 的高度到达屏幕上边缘。

## 修复
- 将 `NoteWindowPage` 的布局对齐从 `bottomCenter` 改为 `topCenter`：
  - Header 隐藏时，多出来的空间落到窗口底部，不再影响贴纸可视内容的上边界；
  - 仍然保留窗口高度中预留的 `Header(32px)`，避免悬浮/编辑时出现挤压/裁切。

## 验证
1) 打开任意贴纸独立窗口。
2) 将窗口拖到屏幕最上方。
3) 预期：贴纸内容可以贴到屏幕顶边（最多保留已有的 `4px`/阴影视觉间距），不再出现 `≈32px` 的透明空隙。
